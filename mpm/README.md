# LP モード純度評価アプリケーション

このアプリケーションは、IR カメラで取得された熱画像に対して画像処理・位相回復計算・モード純度評価を高速に実行します。UKAEA FAT において、平田さん作成の位相回復コードはリアルタイムなモード純度評価ができず、試験に時間がかかってしまったことが開発のきっかけです。

アプリの特徴は以下の通りです。
- GUI アプリで解析のすべてが完結します。
- GUI は Web フロントで、すべて Python で書かれています。余計な環境構築が必要ありません。
- 非常に高速。平田さんコード・QST 中井さんコードの数十倍～数百倍の速度で動作します。
- 位相回復アルゴリズムの改善。アルゴリズムを改良し、局所解への落ち込みを回避しています。

2025/06/13 の GA ジャイロトロン FAT の結果によって、平田さんコードや QST コードより数十倍～数百倍高速に動作し、かつ計算は両コードや GA で使用されている Yuri Gorelov のコードと同様の結果が得られることを検証済です。

- アプリのセットアップや使用方法を知りたい場合、このまま読み進めてください。
- 画像処理（透視変換）の詳細についてはこちらをご覧ください。
- 位相回復計算の数学的な詳細についてはこちらをご覧ください。

## 1. セットアップ

### 1.1. 推奨環境

- 実行環境: WSL2 (Ubuntu 推奨) + ブラウザ
  - Win からでも動かせるはずですが…。
- 使用言語: all Python
  - Python は 3.10 以上のバージョンを推奨
  - （このアプリでは Julia の使用を完全に取りやめました）

### 1.2. インストール

GitHub にリポジトリ `ModePurityMeasurement` がありますので `clone` してください。

```shell
    git clone https://github.com/kyotofusioneering/ModePurityMeasurement.git
```

仮想環境を立ち上げます。

```shell
    cd ModePurityMeasurement
    source .venv/bin/activate
```

依存パッケージを導入します。
```shell
    python3 -m pip install -r requirements-web.txt
```

以下一気に取り込んでいますが、含まれているパッケージは以下の通りです。

- `numpy` ... 数値処理
- `scipy` ... 数値処理
- `matplotlib` ... 描画処理
- `pandas` ... データ処理
- `opencv-python` ... 画像処理
- `nicegui` ... GUI


## 2. 熱画像撮影によるデータの取得

ミリ波ビームのプロファイルは、ミリ波をターゲットに照射したときの温度上昇を赤外線カメラ (IR カメラ) で測定することで得ることができます。ここで

- 温度プロファイル $\boldsymbol{\Delta T}$ は強度プロファイル $\boldsymbol{I} = |\boldsymbol{E}|^2$ に対応すること
- 熱画像からはビームの位相情報を復元することはできないこと（位相問題と呼ばれます）

に留意してください。また KF では、通例 Avio（日本アビオニクス社）の IR カメラを使用して熱画像を撮影しているため、これを前提とします。違うメーカーの IR カメラを使用する場合、関数の追加が必要になります。


### 2.1 IR カメラの遠隔操作と撮影

Avio の IR カメラは附属のアプリ InfRecAnalyzer Pro で遠隔操作とリアルタイムモニタが可能です。以下の手順で熱画像を撮影します。

1. IR カメラと制御 PC を接続し、"Connect" ボタンを押して通信を確立する。リアルタイムモニタができているか確認する。
2. パルス時に録画ボタンを押し、ビームパターンが撮れたら録画を止める。

### 2.2 熱画像の出力

撮像された熱画像は `**.SVX` というファイル名で出力されます。もうひとつのアプリ InfRec Analyzer Standard を使うことで、撮影された熱画像を確認・保存することができます。以下の手順で保存します。

1. 熱画像を読み込み、ビームプロファイルがしっかり直後のフレームを CSV 出力します。このとき以下の通りにしてください。
   1. デリミタは TAB に設定。
   2. ファイル名は距離、撮影日時、フレーム数を盛り込んで `[z]mm_[yyyymmddhhmm]_[frame]f.csv` としてください。
2. ビームプロファイルが見える直前のデータを同様に保存します。
   1. デリミタは TAB。
   2. ファイル名は `[z]mm_[yyyymmddhhmm]_[frame]f_BG.csv` としてください。
3. アプリ上で、ターゲットフレームの四隅にポイントを設定します。
   1. 適切な画像処理のため、四隅は向かって左上から時計回りに選択してください。
   2. なお、カメラはジャイロトロン方向を向いてプロファイルを撮影している、という前提です。
   3. もしカメラとターゲットフレームの相対位置が固定されていれば、コーナー座標を記録したファイルは 1 つあれば十分です。`[yyyymmdd].ppf` として保存してください。
   4. 都度フレームの四隅の座標が変化するようであれば、各距離ごとに `[z]mm_[yyyymmddhhmm].ppf` として保存してください。
4. 保存後、データをまとめてひとつのフォルダに移動し、解析アプリの中の適当なディレクトリに移動します。
```shell
# example
scp /mnt/c/Users/******/beam_data/ ~/ModePurityMeasurement/data/beam_data/
```

ここまでで生成されるデータは、例えば以下のような感じになります。

- `300mm_202506130157_0155f.csv`
- `300mm_202506130157_0150f_BG.csv`
- `300mm_202506130157.ppf`


## 3. GUI アプリによるデータの解析

### 3.1. 起動

```shell
python3 ModePurityWebApp.py
```

起動後、Windows 側ブラウザで `http://localhost:8081` を開きます。ターミナルにリンクが表示されるので、それをクリックしていただくのが楽です。

### 3.2. データの読み込みと画像処理

1. **Data folder path** にデータを集約したフォルダのパスを指定します。
2. **Image size** に、熱画像のフレームサイズを入力します。
3. **PT pixels** に、透視変換後の画像のピクセルサイズを指定します。
   1. 透視変換時に補完するので正直いくつでもよいのですが、のちの FFT を考えて 2 の冪乗かつ熱画像のサイズ (480 x 640) より小さい **256** をデフォルトにしています。
   
![起動画面](./docs/img/README/img1.png)

4. **Load + Preprocess** を押下すると、解析のために以下の画像処理を実施します。
   1. CSV のヘッダーを除去
   2. バックグラウンドデータを差し引いて正味の温度分布データを作成
   3. `*.ppf` の情報から ROI を生成して、領域外はマスク
   4. **透視変換 (perspectove transformation)** によって先ほど指定したピクセル数四方のデータに変換

5. 上記までの手順が終わると、距離 `z` の昇順で変換後の熱画像と各画像の情報を表示します。
   1. z 方向位置、撮影時刻、フレーム番号、ファイル名
   2. ビームの重心（1次モーメント）、幅（2次モーメント）
6. 上記のビームの重心情報から、x-z, y-z 間で線形回帰を計算し、ビームの**傾き**、および **z = 0 mm 位置でのオフセット**を計算します。

![熱画像処理後](./docs/img/README/img2.png)

### 3.3. 位相回復計算

1. 先ほど表示した熱画像の上に **Use for phase retreival** というチェックボックスがあるので、位相回復に使用したい要素だけ選択します。
   1. 5 枚以上選択することを推奨します。最低、位相回復計算の成立のために 3 枚以上は選択してください。
   2. z = 100 ~ 200 mm のデータでは、導波管出口との干渉による擾乱が見られます。これらのデータは選択しないのが良いと思います。
2. Phase Retrieval Settings　を設定します。多くはデフォルト値のままでよいです。

- `Frequency [GHz]`:
  - ミリ波の周波数 $f$。周波数測定を並行して実施しているのであれば、測定値を記入するのが良いです。
  - 伝播計算の波長に影響するので、値を誤ると位相が逸脱して結果が悪化します。
- `Frame size [mm]`:
  - ターゲットを張り付けているフレームの一辺の長さ。
  - 画素あたりの実空間サイズ（`dx_m`）に影響するので、値を誤ると伝播計算の結果がスケール誤差を持ちます。
- `Waveguide radius [mm]`:
  - コルゲート導波管半径。LPモード純度計算で使用。
- `Evaluation z [mm]`:
  - 位相回復後の電場を表示・LPモード純度を評価する基準面の位置。
- `Iterations`:
  - 最大反復回数。Auto stop が ON の場合、条件を満たすと途中で終了します。
- `HIO beta`:
  - HIO のマスク外部分の更新係数（$u = u_\mathrm{prop} - \beta u_\mathrm{new} \ \mathrm{where} \ r \notin \mathbb{M}$）。
  - 目安は $0.6 < \beta < 0.85$ くらいか。大きいと探索は強いが振動しやすく、小さいと安定だが収束が遅い。
- `HIO cycles`, `ER cycles`:
  - 1 計算サイクル内のアルゴリズムの利用回数。`HIO` 回 HIO を実施し、`ER` 回 ER を実施するサイクルを繰り返す。
- `HIO mask percentile`:
  - HIO のマスク閾値です。`a >= percentile(a, p)` をマスク領域 $\mathbb{M}$ として、内側と外側を定義。
- `Auto stop`:
  - 有意改善が止まったときに自動停止するか否か。停止時、「最後」ではなく「最良誤差時点」の解を採用します。
- `Min iter`:
  - `Auto stop` ON 時でも必ず回す最小反復回数。
- `Patience`:
  - 最良値の改善が見られなくても許容する反復数です。`Patience` 連続で有意改善がなければ停止。
- `Rel improve tol`, `Abs improve tol`:
  - 「有意改善」の判定閾値。`best_err - err > max(abs_tol, rel_tol * max(best_err, 1e-12))`。
  - ノイジーなデータでは大きめに設定するほうが良いと思います。
- `LP m max`, `LP n max`:
  - 純度評価する LP モード展開の上限次数。計算量と評価の網羅性のトレードオフになります。
- `Output dir`:
  - 誤差履歴、再構成画像、モード純度ファイル等の保存先。適当に指定してください。

3. **Phase Retrieval** ボタンを押下すると、チェックを付けたデータのみを利用して位相回復計算が実行されます。
   1. 位相回復計算の進行度はプログレスバーで確認できます。

![位相回復計算](./docs/img/README/img3.png)

### 3.4. 結果の表示

1. 位相回復計算が終了すると、結果が表示されます。
   1. プログレスバーの下に、ビーム中心・使用画像枚数・誤差の到達値・停止理由が表示。
   2. `Evaluation z` で決めた位置における位相回復後のビームプロファイル（強度・位相）。
   3. LPmn モード純度。
      1. Axis-centered の値が定義通りの値。Centroid-centered はオフセットが完全になくなった場合の理想値を表示。
      2. モード純度は各列で並び替え可能。**Major** とあるやつだけ見ていれば十分だと思います。

![結果](./docs/img/README/img4.png)

2. 指定したフォルダに、以下のファイルが出力されています。

- `error_history.txt`
  - 誤差改善の履歴。
- `error_curve.png`
  - 誤差改善の履歴を描画したもの。
- `reconstruction.png`
  - 位相回復後のビームプロファイル。
- `mode_purity.txt`
  - LP モード純度評価結果の一覧。
